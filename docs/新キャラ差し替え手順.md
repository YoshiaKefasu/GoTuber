# 新キャラ差し替え手順・注意点

このドキュメントは、GoTuber でキャラクターを差し替える時の完全手順書です。
**元プロジェクト [tomari-guruguru](https://github.com/rotejin/tomari-guruguru) のキャラクターシステムを 100% Go に port したもの**を使用します。

> 設計・ファイル命名・コンポーネント切り出し・スライスツールのアルゴリズム・設定スキーマ・行/列の向き、すべて元プロジェクトと同一です。

---

## 必要な素材

新キャラ用に、以下の 6 枚の画像を用意します。

| シート | 目 | 口 | ファイル名の例 |
|---|---|---|---|
| A | 開け | とじ | `A_目開け_口とじ.png` |
| B | 開け | 中間 | `B_目開け_口中間.png` |
| C | 開け | 開け | `C_目開け_口開け.png` |
| D | 閉じ | とじ | `D_目閉じ_口とじ.png` |
| E | 閉じ | 中間 | `E_目閉じ_口中間.png` |
| F | 閉じ | 開け | `F_目閉じ_口開け.png` |

**条件**:

- **4500×4500px の透過 PNG** (または RGBA)
- 5×5 グリッド、1 コマ **900×900px**
- 行: 上向き → 下向き
  - `r0`: 上を見る
  - `r1`: やや上
  - `r2`: 正面
  - `r3`: やや下
  - `r4`: 下を見る
- 列: 左向き → 右向き
  - `c0`: 左を見る
  - `c2`: 正面
  - `c4`: 右を見る
- A〜F でキャラ位置・角度・体の大きさは同じにする
- 目と口以外はできるだけ同一にする

---

## 重要な注意点

### 1. 900px セルで単純に切らない

素材によっては、髪・ポニテ・服の一部が隣の 900px セルにはみ出すことがあります。

単純に 900×900 で 25 分割すると、次の問題が出ます:

- 右上 / 左上を見るコマに、横のキャラが混入する
- 髪の上やポニテが見切れる
- 下段のキャラ頭部が上段コマに入り込む

そのため、`tools/slice_character_sheets.py` の **component mode** を使います。
これはシート全体からキャラ単位の連結成分を検出して、25 方向へ割り当てる方式です。

### 2. グレー背景の半透明ゴミに注意

元 PNG が透過に見えても、境界付近に半透明のグレー背景が残ることがあります。

症状:

- キャラ周囲に薄い灰色の四角や線が見える
- 黒背景や OBS クロマキー時に境界が汚くなる

スライス時に `--remove-gray-residue` を使って低彩度グレー残りを除去します。

### 3. 行/列の向き (元プロジェクトの規約)

- **行 0 = 上向き**、行 4 = 下向き
- **列 0 = 左向き**、列 4 = 右向き

GoTuber の `internal/mouse/follow.go` は元 `src/app.jsx` と同じ Y 軸反転なしの計算式を使用します。

**元プロジェクトとの参照箇所**:

| 言語 | ファイル | 行 | 式 |
|---|---|---|---|
| JavaScript (元) | `src/app.jsx:61-62` | 61-62 | `clamp(Math.round((current+1) / 2 * (N - 1)), 0, N - 1)` |
| Go (GoTuber) | `internal/mouse/follow.go:60-69` | 60-69 | `int(math.Round((f.current + 1) / 2 * 4))` |

両方とも `r0=上、r4=下` で **コーナー (-1, 0, 1) と中点 (±0.5) で完全一致**。
JS は `Math.round`、`* (N-1) = * 4` で 0.5 ステップ幅の最近接丸め、Go は `int(math.Round(...))` で同じ動作（`test/follow_test.go:TestFollow_Granularity` で 5 中心点 -1, -0.5, 0, 0.5, 1 の完全一致を検証）。

## 生成手順

例: 新素材フォルダが `新キャラ資料` の場合。

### 1. 依存ツールのインストール

```bash
# ffmpeg / ffprobe (必須、GoTuber のスライスツールが呼ぶ)
# macOS
brew install ffmpeg
# Ubuntu / Debian
sudo apt install ffmpeg
# Windows
# https://www.gyan.dev/ffmpeg/builds/ から zip を DL し、PATH に追加

# Python 依存 (テスト用 pytest のみ、Python パッケージは不要)
pip install -r tools/requirements.txt
```

### 2. スライス実行

```bash
python tools/slice_character_sheets.py \
  --source "新キャラ資料" \
  --slices-out "assets/characters/mychara" \
  --format webp \
  --jobs 4 \
  --component-mode \
  --min-component-area 80 \
  --alpha-threshold 64 \
  --remove-gray-residue
```

### 3. 出力

```
assets/characters/mychara/A/r0c0.webp ... r4c4.webp
assets/characters/mychara/B/r0c0.webp ... r4c4.webp
...
assets/characters/mychara/F/r0c0.webp ... r4c4.webp
```

合計 150 枚 (6 シート × 5×5 グリッド)、各画像 1200×1200px (anchored at 600, 900)。

## 設定 (GoTuber)

`config/mychara.yaml` を作成。**`src/character-config.js` と完全互換のキー名** (camelCase) を使用します。

```yaml
# GoTuber キャラクター設定 (character-config.js 互換)
basePath: "assets/characters/mychara"
ext: "webp"  # または "png"
rows: 5
cols: 5
sheets:
  eyesOpen:
    close: "A"
    half: "B"
    open: "C"
  eyesClosed:
    close: "D"
    half: "E"
    open: "F"
```

**起動**:

```powershell
# 1. 既存 config/default.yaml の basePath を変更する (最も簡単)
#    default.yaml を開いて basePath を書き換え

# 2. または、新しい設定ファイルを作って main.go に渡す (将来対応予定)
#    ※ Phase 1.12 時点では main.go は config/default.yaml を直接読む。
#    ※ 複数キャラを使い分ける場合は default.yaml を切り替える運用。
```

`character-config.js` 互換のキー名 (`basePath`、`eyesOpen`/`eyesClosed`、`close`/`half`/`open`)。

## 生成ログで見るポイント

各シートで以下のように出ていれば正常です:

```text
A: A_目開け_口とじ.png (150 pending)
  components=xxxx large=25
  row0 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
  row1 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
  ...
  row4 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
```

確認ポイント:

- `large=25` になっている
- 各行の `comps(area)` が全部 `1:xxxxx`
  - 1 セルに 1 キャラ成分だけ割り当たっている状態

NG 例:

- `large=24` や `large=26`
- どこかが `0:0`
- どこかが `2:xxxxx`

この場合は切り抜き検出が失敗している可能性があります。

調整候補:

1. **Best**: `--alpha-threshold` を調整する
   - ゴミが多い: `80` や `96`
   - 細い髪が消える: `32` や `48`
2. **Second-best**: `--min-component-area` を調整する
   - 小物や髪束が別成分になる場合は小さめ
   - ノイズを拾う場合は大きめ
3. **Not recommended**: `--no-component-mode` で単純セル切り
   - 隣キャラ混入・髪見切れが再発しやすい

## 必ずやる検証

### 1. 枚数確認

```bash
# Linux / macOS
for s in A B C D E F; do
  count=$(ls "assets/characters/mychara/$s" | wc -l)
  echo "$s files=$count"
done

# PowerShell
foreach ($s in 'A','B','C','D','E','F') {
  $count = (Get-ChildItem -Path (Join-Path 'assets/characters/mychara' $s) | Measure-Object).Count
  Write-Output "$s files=$count"
}
```

全て `25` なら OK です。

### 2. 代表コマを目視確認

最低限見るコマ:

```
assets/characters/mychara/A/r0c0.webp   左上を見る
assets/characters/mychara/A/r0c4.webp   右上を見る
assets/characters/mychara/A/r2c2.webp   正面
assets/characters/mychara/F/r0c0.webp   目閉じ口開け・左上
assets/characters/mychara/F/r0c4.webp   目閉じ口開け・右上
assets/characters/mychara/F/r2c2.webp   目閉じ口開け・正面
```

見るポイント:

- 横のキャラが混ざっていない
- 髪の上、ポニテ、耳飾りが切れていない
- 足元 / 服の下が変に切れていない
- 灰色の四角や背景残りがない
- A〜F で位置がズレていない

### 3. 寸法確認

```bash
ffprobe -v error -select_streams v:0 -show_entries stream=width,height -of csv=p=0:s=x \
  assets/characters/mychara/A/r2c2.webp
# → 1200x1200 なら OK
```

## アプリ側の確認

スライス画像の参照先は `config/default.yaml` (または `config/*.yaml`) で一元管理しています。

新キャラ差し替え時は、`config/default.yaml` の `basePath` を変更するだけで OK です。

実行確認:

```powershell
# 開発モード (F1 で Tweaks パネル、Esc で終了)
.\scripts\dev.ps1

# リリースビルド
.\scripts\build.ps1
.\bin\gotuber.exe
```

確認ポイント:

- マウス追従で 25 方向が自然に切り替わる
- 左上 / 右上方向で横のキャラが出ない
- 口パク時に A/B/C、D/E/F の位置が跳ねない
- まばたき時に顔位置がズレない
- F1 で Tweaks パネルが開き、Auto Blink ON/OFF で D/E/F に切替わる
- 喋ると口パク (A/B/C or D/E/F) する

## 次回の推奨フロー

1. 新キャラ 6 シートを専用フォルダに入れる
2. `ffprobe` で 4500×4500 RGBA か確認
3. `tools/slice_character_sheets.py` を component mode で実行
4. ログで `large=25` と各セル `1:xxxxx` を確認
5. A/F の 5×5 全体を目視確認
6. 数値検査で中心 X / 足元 Y / 端接触を確認
7. `config/default.yaml` の `basePath` を新しいスライスディレクトリに変更
8. `.\bin\gotuber.exe` で実動作確認

この順番でやると、切り抜きミスを早い段階で発見できます。

## 元プロジェクトとの対応表

| 概念 | 元 (tomari-guruguru) | GoTuber (Phase 1.12) |
|---|---|---|
| 設定ファイル | `src/character-config.js` | `config/*.yaml` (YAML, camelCase キー) |
| 設定読み込み | `import charConfig from './character-config'` | `character.LoadConfig(path)` |
| 設定構造 | `basePath`, `ext`, `rows`, `cols`, `sheets.{eyesOpen,eyesClosed}.{close,half,open}` | 同上 |
| パス生成 | `charConfig.src(sheet, r, c)` | `cfg.Src(sheet, r, c)` |
| シート取得 | `charConfig.sheets.eyesOpen.close` etc. | `cfg.SheetFor(eyesClosed, mouth)` |
| 画像ディレクトリ | `public/slices2/{A-F}/` | `assets/characters/<name>/{A-F}/` |
| ファイル名 | `r{0-4}c{0-4}.{ext}` | 同じ |
| マウス追従 | `app.jsx:60-62` Y 軸反転なし | `mouse/follow.go` Y 軸反転なし |
| スライスツール | `tools/slice_character_sheets.py` (Python) | **同名** (Python, MIT 継承) |
